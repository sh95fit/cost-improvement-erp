# 04_전체 구현 체크리스트 및 코드기준안.md

## 문서 목적

이 문서는 현재 프로젝트의 구현 검증판이다.
무엇을 수정해야 하는지, 무엇이 완료로 인정되는지, 어떤 항목이 금지인지, 어떤 파일을 어떤 순서로 건드려야 하는지를 체크리스트 형태로 정리한다.

이 문서는 현재 구조 재정의 라운드에 초점을 맞춘다.
전체 히스토리는 `PROGRESS.md`,
구조 기준은 `02_개발문서.md`,
실행 절차는 `03_개발가이드문서.md`,
변경 이력은 `05_불일치 정리 및 통합기준 제안.md`를 따른다.

## 1. 공통 코드 기준

### 타입
- [ ] `any` 사용 없음
- [ ] 불가피한 경우 `unknown` 후 narrowing 사용
- [ ] Prisma 타입 직접 활용 가능하면 커스텀 타입 최소화

### 로깅
- [ ] `console.log` 없음
- [ ] `logger` 사용

### DB 접근
- [ ] 클라이언트 컴포넌트에서 DB 직접 접근 없음
- [ ] action 또는 API를 통해서만 접근

### soft-delete
- [ ] where 조건 누락 없음
- [ ] explicit soft-delete 또는 extension 정책 일관성 유지

### 트랜잭션
- [ ] 다중 쓰기 작업은 transaction 적용
- [ ] copy / cascade delete / multi-create 경로 점검

### 권한
- [ ] 모든 변경 action에서 permission 검사
- [ ] 회사/범위 검증 유지

### 감사 로그
- [ ] create/update/delete 주요 작업에 audit log 기록

### 테스트
- [ ] 서비스 변경 시 테스트 동반
- [ ] 회귀 테스트 확인

### 문서
- [ ] `PROGRESS.md` 갱신
- [ ] 구조 변경 시 `05_불일치 정리 및 통합기준 제안.md` 갱신
